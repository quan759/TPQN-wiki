THQN Algorithm Explorer

Độ phức tạp của thuật toán (Algorithm Complexity) là một khái niệm quan trọng trong lập trình thi đấu. Nó đánh giá hiệu suất của một thuật toán dựa trên sự phụ thuộc của thời gian và không gian thực thi vào kích thước của đầu vào (input size). Độ phức tạp giúp đo lường sự tăng trưởng của tài nguyên (thời gian hoặc bộ nhớ) cần thiết khi kích thước của đầu vào tăng lên.



Có hai loại chính của độ phức tạp thuật toán:



  • Độ phức tạp thời gian (Time Complexity): Đo lường thời gian thực thi của thuật toán khi kích thước của đầu vào tăng lên. Thời gian thực thi thường được đo bằng số bước thực thi (so sánh, gán, tính toán, v.v.) mà thuật toán thực hiện trên một máy tính.


  • Độ phức tạp không gian (Space Complexity): Đo lường lượng bộ nhớ cần thiết để thực thi thuật toán khi kích thước của đầu vào tăng lên. Độ phức tạp không gian thường được đo bằng số lượng ô nhớ (bit, byte, word, v.v.) mà thuật toán sử dụng.


    • Ký hiệu thường được sử dụng để biểu thị độ phức tạp là "\(O\)" (Big \(O\) notation). Độ phức tạp thời gian và không gian được đánh giá trong trường hợp tốt nhất, trung bình và xấu nhất, nhưng thường ta quan tâm đến trường hợp xấu nhất (worst-case complexity) vì nó đảm bảo hiệu suất của thuật toán trong mọi trường hợp.



    Một số loại độ phức tạp thời gian phổ biến và ví dụ của chúng:



  • Độ phức tạp \(O(1)\) - Hằng số: Thời gian thực thi không phụ thuộc vào kích thước đầu vào.


    • void printFirstElement(int arr[]) {
    cout << arr[0] << endl;
}


  • Độ phức tạp \(O(\log n)\) - Logarithmic: Thời gian thực thi tăng theo logarit của kích thước đầu vào.
    • int binarySearch(int arr[], int n, int target) {
    int left = 0;
    int right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}


  • Độ phức tạp \(O(n)\) - Tuyến tính: Thời gian thực thi tăng tuyến tính với kích thước đầu vào.
    • int sumArray(int arr[], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++) {
        sum += arr[i];
    }
    return sum;
}


  • Độ phức tạp \(O(n^2)\) - Bậc hai: Thời gian thực thi tăng theo bậc hai của kích thước đầu vào.


    • void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr[j], arr[j + 1]);
            }
        }
    }


    Độ phức tạp của thuật toán là một yếu tố quan trọng để xác định thuật toán nào sẽ hoạt động hiệu quả với các kích thước đầu vào khác nhau. Trong lập trình thi đấu, việc hiểu và đánh giá độ phức tạp của thuật toán giúp bạn tối ưu hóa chương trình của mình và tìm ra các giải pháp tối ưu cho các bài toán có độ khó lớn.
